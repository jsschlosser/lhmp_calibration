Configuring the Raspberry Pi

This section summarizes the steps required to setup the environment needed to run a GigE Vision (i.e., genicam) camera on a Raspberry Pi 4b. Here we use a Pheonix 5.0 Polarization camera (LUCID Vision Labs Inc., 2023) that is built around Sony’s IMX250MYR CMOS.

Hardware Requirements
  1. Any GigE vision camera should work however it must be a GenICam compliant machine vision camera/device.

    1. Power supply.

  2. Rasberry Pi 4b running Debian GNU/Linux 12 (bookworm) x64. Prefer remote connection with Raspberry Pi Connect.

    1. Power supply.

  3. Ethernet Cable.

Setup Hardware

  1. Connect camera to Raspberry Pi via ethernet cable.

  2. Power Raspberry Pi.

Install ArenaSDK Software (ArenaSDK_v0.1.78_Linux_ARM64)

Note

Full details in ‘README_ARM64.txt’

  1. Set jumbo frames
    $ sudo ip link set enp0s8 mtu 9000
  2. Set receive buffers
    $ sudo ethtool -g enp0s8
$ sudo ethtool -G enp0s8 rx 4096
  3. Set socket buffer size
    $ sudo sh -c "echo 'net.core.rmem_default=33554432' >> /etc/sysctl.conf"
$ sudo sh -c "echo 'net.core.rmem_max=33554432' >> /etc/sysctl.conf"
$ sudo sysctl -p
  4. Reverse path filtering
    $ sudo sh -c "echo 'net.core.rmem_default=33554432' >> /etc/sysctl.conf"
$ sudo sh -c "echo 'net.core.rmem_max=33554432' >> /etc/sysctl.conf"
$ sudo sysctl -p
  5. Extract the tarball to your desired location:
    $ tar -xvzf ArenaSDK_Linux_ARM64.tar.gz
  6. Run the ArenaSDK_Linux_ARM64.conf file
    $ cd ~/Documents/ArenaSDK_v0.1.78_Linux_ARM64/ArenaSDK_Linux_ARM64
$ sudo sh Arena_SDK_ARM64.conf
Install arena api

Note

Full details in ‘~/arena_api-2.7.1-py3-none-any/README.txt’

  1. Setup virtual environment.
    $ python3 -m venv myvirtualenv
  2. Activate virtual environment.
    $ source ~/myvirtualenv/bin/activate
  1. Install standard openCV.
    $ pip install opencv-python
  1. Install arena api.
    $ cd arena_api-2.7.1-py3-none-any
$ pip install arena_api-2.7.1-py3-none-any.whl
$ pip install -r examples/requirements_lin_arm64.txt
$ sudo apt-get install python3-tk

Example of running programs in Arena api

  1. Activate virtual environment.
    $ source ~/myvirtualenv/bin/activate
  2. Set appropriate directory with python scripts.
    $ cd ~/Documents
  3. Run desired python script.
    $ python3
$ import TestSample
$ TestSample.Run()

Instructions for building sphinx documentation locally

This section describes how to build the sphinx documentation locally.

  1. Activate virtual environment.
    $ source ~/myvirtualenv/bin/activate
  2. Install matplotlib.
    $ pip install matplotlib
  3. Install basic sphinx package.
    $ pip install sphinx
  4. Install html theme for sphinx.
    $ pip install sphinx_rtd_theme
  5. Install pdf builder for sphinx.
    $ pip install sphinx-simplepdf
  6. Build sphinx.
    $ sphinx-build -b html source docs

Test the instrument functionality

TestSample.Run()

Function for inputting camera settings and acquiring images from the LHMP. Writes output level 0 data to netCDF file for processing.

Raw_Capture.Run(camera_settings)

Function for capturing samples with the LHMP.

Parameters:

camera_settings (numpy dictionary) – dictionary containing the gain and exposure time camera settings as well as the acquisition duration

Returns:

dictionary of raw image data in digital number along with the image metadata of exposure time (us), gain, acquisition time (UTC), sensor temperature (degC)

Return type:

numpy dictionary

raw_data_file_gen.Run(raw_data_dict, output_filename)

Function for capturing samples with the LHMP.

Parameters:

  • raw_data_dict (numpy dictionary) – raw image dictionary with ancillary data.

  • output_filename (string) – user set output path/filename.

TestPlot.standard_test()

Function for visualizing standard polarizer DoLP and AoLP.

TestPlot.demosaic_test()

Function for demosaicing and visualizing DoLP, AoLP, intensity, and the Stoke’s vector components.

Perform dark calibration measurements

Dark_cal_data_collection.DarkCurrent()

Function for capturing a set of dark current measurements by running this on starup with the instrument with the lense cap on.

Dark_cal_data_collection.DarkRead()

Function for capturing a set of dark read measurements by varying expsure time after running the DarkCurrent routine.

Visualize dark calibration measurements

Dark_cal_data_processing.DarkCurrent(pathto_raw_dark_cal_data=None)

Takes LHMP dark measurements (in DN) and plots them against temparature. Requires numpy, matplotlib, and pylab.

Parameters:

pathto_raw_dark_cal_data (string) – Path to dark calibration dataset.

Dark_cal_data_processing.DarkRead(pathto_raw_dark_cal_data=None)

Takes LHMP dark measurements (DN) and plots them against exposure time (us). Requires numpy, matplotlib, and pylab.

Parameters:

pathto_raw_dark_cal_data (string) – Path to dark calibration dataset.
